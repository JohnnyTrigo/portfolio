01 · O desafio

O problema que
esse projeto resolve

Times de operações de telecomunicações gerenciam centenas de demandas mensais — atualizações de tabelas de preços, configurações de produto, relatórios executivos — sem nenhuma visibilidade centralizada de performance.

Os dados existem, mas estão em silos: planilhas, e-mails, Planner, sistemas legados. Sem integração, sem histórico, sem SLA monitorado em tempo real.

01
Dados em silos

Informações espalhadas entre Planner, Forms e planilhas Excel sem integração. Nenhuma fonte central da verdade. A equipe gasta horas compilando relatórios manualmente.

02
SLA invisível

Sem monitoramento de lead time ou aderência a SLA em tempo real. Violações descobertas somente no relatório mensal — tarde demais para ação corretiva.

03
Retrabalho oculto

Taxa de correções não rastreada por categoria ou analista. Impossível identificar padrões de qualidade, gargalos e oportunidades de melhoria de processo.

-- ── Q1: KPI Executivo por Mês ─────────────────────────────────────────────
-- Responde: Volume · Aderência SLA · Retrabalho · Lead Time
SELECT
    d.nr_ano,
    d.nm_mes,
    COUNT(*)                                                AS Total_Solicitacoes,
    SUM(CASE WHEN s.fl_dentro_sla = 1 THEN 1 ELSE 0 END)  AS Dentro_SLA,
    SUM(CASE WHEN s.fl_correcao   = 1 THEN 1 ELSE 0 END)  AS Com_Correcao,
    ROUND(
        100.0 * SUM(CASE WHEN s.fl_dentro_sla = 1 THEN 1 ELSE 0 END)
              / NULLIF(SUM(CASE WHEN s.fl_dentro_sla IS NOT NULL
                               THEN 1 ELSE 0 END), 0),
    2)                                                      AS Pct_Aderencia_SLA,
    ROUND(
        100.0 * SUM(CASE WHEN s.fl_correcao = 1 THEN 1 ELSE 0 END)
              / NULLIF(COUNT(*), 0),
    2)                                                      AS Pct_Retrabalho,
    ROUND(AVG(s.nr_lead_time_horas), 1)                    AS Lead_Time_Medio_h
FROM ops.fct_solicitacoes  s
JOIN ops.dim_data          d  ON s.sk_data_abertura = d.sk_data
GROUP BY d.nr_ano, d.nr_mes, d.nm_mes
ORDER BY d.nr_ano, d.nr_mes;
-- ── Q2: Ranking de Analistas — Score Composto ─────────────────────────────
-- Score = 70% Aderência SLA + 30% Qualidade (inverso do retrabalho)
SELECT
    a.nm_analista,
    a.nm_squad,
    a.nm_nivel,
    COUNT(*)                                                           AS Total,
    ROUND(AVG(s.nr_lead_time_horas), 1)                                AS Lead_Time_h,
    ROUND(
        100.0 * SUM(CASE WHEN s.fl_dentro_sla = 1 THEN 1 ELSE 0 END)
              / NULLIF(SUM(CASE WHEN s.fl_dentro_sla IS NOT NULL
                               THEN 1 ELSE 0 END), 0),
    1)                                                                 AS Pct_SLA,
    ROUND(
        100.0 * SUM(CASE WHEN s.fl_correcao = 1 THEN 1 ELSE 0 END)
              / NULLIF(COUNT(*), 0),
    1)                                                                 AS Pct_Retrabalho,
    -- Score ponderado: 70% SLA · 30% qualidade
    ROUND(
        (100.0 * SUM(CASE WHEN s.fl_dentro_sla = 1 THEN 1 ELSE 0 END)
               / NULLIF(SUM(CASE WHEN s.fl_dentro_sla IS NOT NULL
                               THEN 1 ELSE 0 END), 0)) * 0.7
      + (1 - (100.0 * SUM(CASE WHEN s.fl_correcao = 1 THEN 1 ELSE 0 END)
                         / NULLIF(COUNT(*), 0)) / 100) * 100 * 0.3,
    1)                                                                 AS Score
FROM ops.fct_solicitacoes  s
JOIN ops.dim_analista      a  ON s.sk_analista = a.sk_analista
WHERE s.nm_status = 'Concluída'
GROUP BY a.nm_analista, a.nm_squad, a.nm_nivel
ORDER BY Score DESC;
-- ── Q3: Tendência com Média Móvel 3M + Variação YoY ──────────────────────
-- Suaviza sazonalidade e compara com mesmo período do ano anterior
WITH base AS (
    SELECT
        d.nr_ano, d.nr_mes, d.nm_mes,
        COUNT(*)                                                   AS total,
        SUM(CASE WHEN s.fl_dentro_sla = 1 THEN 1.0 ELSE 0 END)
          / NULLIF(SUM(CASE WHEN s.fl_dentro_sla IS NOT NULL
                           THEN 1.0 ELSE 0 END), 0)                AS pct_sla,
        AVG(s.nr_lead_time_horas)                                  AS lead_time
    FROM ops.fct_solicitacoes s
    JOIN ops.dim_data         d ON s.sk_data_abertura = d.sk_data
    WHERE s.nm_status = 'Concluída'
    GROUP BY d.nr_ano, d.nr_mes, d.nm_mes
)
SELECT
    nr_ano, nm_mes,
    total                                                          AS Volume,
    ROUND(pct_sla * 100, 1)                                        AS Pct_SLA,
    -- Média móvel 3 meses (janela deslizante)
    ROUND(AVG(CAST(total AS FLOAT))
          OVER (ORDER BY nr_ano, nr_mes
                ROWS BETWEEN 2 PRECEDING AND CURRENT ROW), 1)      AS MM3_Volume,
    ROUND(AVG(pct_sla * 100)
          OVER (ORDER BY nr_ano, nr_mes
                ROWS BETWEEN 2 PRECEDING AND CURRENT ROW), 1)      AS MM3_Pct_SLA,
    -- Crescimento YoY via LAG de 12 períodos
    total - LAG(total, 12) OVER (ORDER BY nr_ano, nr_mes)          AS Var_YoY,
    ROUND(100.0
        * (total - LAG(total, 12) OVER (ORDER BY nr_ano, nr_mes))
        / NULLIF(LAG(total, 12) OVER (ORDER BY nr_ano, nr_mes), 0),
    1)                                                             AS Pct_YoY
FROM base
ORDER BY nr_ano, nr_mes;
-- ── Q4: Detecção de Anomalias via Z-Score ─────────────────────────────────
-- Z > 2.0 = Anomalia · Z > 1.5 = Atenção · identifica gargalos ocultos
WITH stats AS (
    SELECT
        sk_categoria,
        AVG(nr_lead_time_horas)    AS media,
        STDEV(nr_lead_time_horas)  AS desvio
    FROM ops.fct_solicitacoes
    WHERE nm_status = 'Concluída'
    GROUP BY sk_categoria
)
SELECT
    s.cd_solicitacao,
    a.nm_analista,
    c.nm_categoria,
    s.nr_lead_time_horas                                           AS Lead_Time_Real,
    ROUND(st.media,  1)                                            AS Media_Categoria,
    ROUND(st.desvio, 1)                                            AS Desvio_Padrao,
    ROUND((s.nr_lead_time_horas - st.media)
        / NULLIF(st.desvio, 0), 2)                                 AS Z_Score,
    CASE
        WHEN (s.nr_lead_time_horas - st.media)
           / NULLIF(st.desvio, 0) > 2.0  THEN '🔴 ANOMALIA'
        WHEN (s.nr_lead_time_horas - st.media)
           / NULLIF(st.desvio, 0) > 1.5  THEN '🟡 ATENÇÃO'
        ELSE '🟢 Normal'
    END                                                            AS Status
FROM ops.fct_solicitacoes  s
JOIN ops.dim_analista      a  ON s.sk_analista  = a.sk_analista
JOIN ops.dim_categoria     c  ON s.sk_categoria = c.sk_categoria
JOIN stats                 st ON s.sk_categoria = st.sk_categoria
WHERE s.nm_status = 'Concluída'
  AND (s.nr_lead_time_horas - st.media)
    / NULLIF(st.desvio, 0) > 1.5
ORDER BY Z_Score DESC;
-- ── Q5: View otimizada para Power BI ─────────────────────────────────────
-- Fonte única da verdade · Suporta DirectQuery e Import Mode
CREATE OR ALTER VIEW ops.vw_solicitacoes_pbi AS
SELECT
    s.cd_solicitacao,
    -- Datas
    da.dt_data       AS dt_abertura,
    da.nr_ano,  da.nr_mes,  da.nm_mes AS nm_mes_abertura,
    da.nm_trimestre,
    df.dt_data       AS dt_fechamento,
    -- Analista
    a.nm_analista,  a.nm_squad,  a.nm_nivel,
    -- Categoria
    c.nm_grupo,  c.nm_categoria,
    c.nr_sla_horas,  c.fl_critico,
    -- Produto
    p.nm_linha,  p.nm_segmento,  p.nm_produto,
    -- Fatos
    s.nm_status,  s.nm_prioridade,  s.nm_canal_entrada,
    s.nr_lead_time_horas,
    s.fl_dentro_sla,  s.fl_correcao,
    s.vl_impacto_receita,
    -- Colunas calculadas prontas para DAX (evitam medidas repetitivas)
    CASE WHEN s.fl_dentro_sla = 1
         THEN 'Dentro do SLA'   ELSE 'Fora do SLA'    END  AS status_sla,
    CASE WHEN s.fl_correcao = 1
         THEN 'Com Retrabalho'  ELSE 'Sem Retrabalho'  END  AS status_correcao,
    CASE WHEN s.nm_status IN ('Aberta','Em Andamento')
         THEN 1 ELSE 0 END                                   AS fl_aberta,
    ROUND(s.nr_lead_time_horas / 8.0, 1)                     AS lead_time_dias
FROM ops.fct_solicitacoes  s
JOIN ops.dim_data          da ON s.sk_data_abertura   = da.sk_data
LEFT JOIN ops.dim_data     df ON s.sk_data_fechamento = df.sk_data
JOIN ops.dim_analista       a ON s.sk_analista         = a.sk_analista
JOIN ops.dim_categoria      c ON s.sk_categoria        = c.sk_categoria
JOIN ops.dim_produto        p ON s.sk_produto          = p.sk_produto;
04 · Business Intelligence

DAX que responde
perguntas de negócio

Cada medida foi projetada para responder uma pergunta executiva específica — não apenas mostrar dados. RLS por perfil + Bookmarks para navegação contextual entre Liderança, Gestores e Analistas.

% Aderência SLAKPI
% Aderência SLA = DIVIDE( CALCULATE( COUNTROWS(fct_solicitacoes), fct_solicitacoes[fl_dentro_sla] = 1 ), CALCULATE( COUNTROWS(fct_solicitacoes), NOT ISBLANK(fct_solicitacoes[fl_dentro_sla]) ), 0 )
Exclui demandas em aberto do denominador — sem isso, o KPI seria distorcido pelo volume não finalizado.
Lead Time MédioMEASURE
Lead Time Médio = CALCULATE( AVERAGEX( FILTER( fct_solicitacoes, NOT ISBLANK( fct_solicitacoes[nr_lead_time_horas] ) ), fct_solicitacoes[nr_lead_time_horas] ) )
AVERAGEX + FILTER garante que demandas ainda abertas não distorçam a média do tempo de atendimento.
Var SLA MoMTREND
Var SLA MoM = VAR atual = [% Aderência SLA] VAR anterior = CALCULATE( [% Aderência SLA], DATEADD( dim_data[dt_data], -1, MONTH ) ) RETURN DIVIDE(atual - anterior, anterior, 0)
Alimenta o ▲▼ no card de SLA. Usa DATEADD na tabela de datas — mais robusto que cálculo direto na fato.
Taxa RetrabalhoQUALITY
Taxa Retrabalho = DIVIDE( CALCULATE( COUNTROWS(fct_solicitacoes), fct_solicitacoes[fl_correcao] = 1 ), COUNTROWS(fct_solicitacoes), 0 ) -- Threshold de alerta: 5% -- Formatação condicional no card
KPI de qualidade. Formatação condicional muda o card para vermelho quando ultrapassa 5% de retrabalho.
Impacto Receita R$FINANCIAL
Impacto Receita = SUMX( FILTER( fct_solicitacoes, NOT ISBLANK( fct_solicitacoes[vl_impacto_receita] ) && fct_solicitacoes[nm_status] = "Concluída" ), fct_solicitacoes[vl_impacto_receita] )
Soma apenas demandas concluídas com impacto registrado. Permite corte por segmento e linha de produto.
Score PerformanceCOMPOSITE
Score Performance = VAR sla = [% Aderência SLA] * 100 VAR qual = (1 - [Taxa Retrabalho]) * 100 RETURN ROUND( sla * 0.7 + qual * 0.3, 1 ) -- 70% SLA · 30% Qualidade
Score composto para ranking de analistas. Alimenta o visual de tabela com formatação condicional por faixa.

Queries de produção

T-SQL com window functions (LAG, AVG OVER), CTEs, Z-Score para anomalias, análise YoY e view otimizada para Power BI. Nenhuma query usa subconsultas aninhadas desnecessárias.
